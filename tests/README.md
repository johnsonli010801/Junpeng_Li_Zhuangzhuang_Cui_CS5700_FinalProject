## 安装和运行

### 1. 安装依赖

```bash
cd tests
npm install
```

### 2. 确保服务器运行

测试前确保后端服务器正在运行：

```bash
cd /root/YouChat
npm run dev
```

服务器应该在以下端口运行：
- 后端: http://localhost:4000
- 前端: http://localhost:5173

### 3. 运行测试

```bash
# 运行完整测试套件
npm test

# 或者直接运行
node comprehensive-blackbox-test.js
```

### 4. 查看测试报告

测试完成后，报告会保存在：
- `test-report.json` - JSON格式详细报告
- `test-output.log` - 控制台输出日志

## 测试配置

可以通过环境变量配置测试：

```bash
# 自定义API地址
API_BASE=http://localhost:4000/api npm test

# 自定义Socket地址
SOCKET_URL=http://localhost:4000 npm test
```

## 测试详情

### 自动化测试流程

1. **用户创建**：创建3个测试用户（A、B、C）
2. **用户登录**：所有用户登录获取token
3. **MFA设置**：为用户A启用MFA
4. **好友关系**：
   - A → B: 发送并接受
   - A → C: 发送但拒绝
5. **会话创建**：创建群聊和私聊
6. **消息测试**：发送和接收消息
7. **文件测试**：上传和下载文件
8. **实时通信**：Socket.IO消息和信令

### 测试数据

- **测试用户**：自动生成唯一邮箱（基于时间戳）
- **测试文件**：自动创建和清理
- **测试会话**：自动创建和管理
- **隔离性**：每次运行使用新的测试数据

## 已知问题

### 需要修复的测试 (3个)

1. **非成员查看消息权限**
   - 测试：非成员不能查看消息
   - 状态：失败
   - 原因：用户C后来被添加到会话中

2. **非成员上传文件权限**
   - 测试：非成员不能上传文件
   - 状态：失败
   - 原因：同上

3. **非成员下载文件权限**
   - 测试：非成员不能下载文件
   - 状态：失败
   - 原因：同上

**解决方案**：测试脚本需要创建一个独立的会话，不添加用户C作为成员。

## 扩展测试

### 添加新测试

在 `comprehensive-blackbox-test.js` 中添加新的测试函数：

```javascript
async function testNewFeature() {
  log('\n========== 新功能测试 ==========', 'info');
  
  await runTest('测试名称', async () => {
    const response = await axios.get(`${API_BASE}/new-endpoint`);
    if (response.status !== 200) throw new Error('测试失败');
  });
}
```

然后在 `runAllTests()` 中调用：

```javascript
await testNewFeature();
```

      - name: Wait for server
